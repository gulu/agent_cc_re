# 四阶段实施路线图

---

## Phase 1：GPU 管线搭建 + 快速止血（1-2 周）

| # | 任务 | 产出 | 工时 |
|---|------|------|:--:|
| 1 | 部署 RTX 4090 环境 (CUDA 12.x + cuDNN) | GPU 可用 | 1 天 |
| 2 | QCService + ReportQC 合并为 Agent_QC | 单一服务 :5100 | 1.5 天 |
| 3 | MedBERT-Chinese ONNX 导出 + GPU 验证 | ONNX 模型 + C# 加载 | 1.5 天 |
| 4 | 搭建 vLLM 本地服务 (Qwen2.5-7B) | vLLM 运行在 :8100 | 1 天 |
| 5 | 知识库 YAML 外部化 | YAML 文件替代 DbSeed 硬编码 | 1.5 天 |
| 6 | NLP 预处理层（否定检测 + 段落解析） | NegationDetector + SectionParser | 2 天 |
| 7 | 快速止血清单 6 项执行 | 规则引擎加固 | 3 天 |
| 8 | 200 份报告基准测试（准确率基线） | 误报率/精确率/召回率基线 | 1 天 |
| 9 | **里程碑：延迟基准测试** | **确认端到端 P95 < 300ms** | — |

**Phase 1 的前置条件：**
- RTX 4090 硬件就位，CUDA 12.4+ 驱动安装完成
- 从 `QcReport` 表脱敏导出 ≥ 200 份历史报告（含医生标注的质控结果）作为基线测试集

## Phase 2：模型微调 + Skill Squad 初版（3-4 周）

**数据准备：**
- 数据来源优先级：(1) 历史 QcReport 表脱敏导出 (2) 公开中文放射科报告数据集（如 CHIP、CBLUE）(3) 人工构造边界 case
- 标注规范：双人独立标注，标注一致性 κ ≥ 0.85；不一致由 Domain Consultant 裁决
- 数据版本管理：使用 DVC 或 Git LFS

| # | 任务 | 产出 | 工时 |
|---|------|------|:--:|
| 1 | 收集/标注放射科报告语料 (≥ 500 份) | 标注数据集 | 5 天 |
| 2 | MedBERT 医学分词器定制 (Jieba + 放射科词典) | 专业分词器 | 2 天 |
| 3 | 错别字检测 Fine-Tuning（序列标注） | 微调模型 | 3 天 |
| 4 | NER 实体识别 Fine-Tuning | 微调模型 | 2 天 |
| 5 | NLI 所见-结论一致性 Fine-Tuning | 微调模型 | 3 天 |
| 6 | Hermes Orchestrator 调度逻辑 | 按需触发机制 | 2 天 |
| 7 | 3 个核心 Skill 编写 + 测试 | gender / critical / nli | 3 天 |
| 8 | **里程碑：200+ 报告准确率评估** | **精确率/召回率/F1 基准** | — |

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|:--:|:--:|------|
| MedBERT 微调后精度不达预期 | 中 | 高 | 准备 PCL-MedBERT 作为备选 |
| vLLM 与 C# 集成问题 | 低 | 中 | 准备 llama.cpp HTTP Server 回退方案 |
| 标注数据量不足 | 中 | 中 | 使用公开 CHIP 数据集补充 |
| GPU 驱动/CUDA 兼容问题 | 低 | 中 | 提前验证环境，锁定 CUDA 版本 |
