# 五层防御体系设计

> 逐层解决五种语境盲区。不是修修补补加 if-else，而是建立递进式防御纵深。

---

## 防御层级总览

```
┌─────────────────────────────────────────┐
│ 第五层：Hermes QA 仲裁 (GPU)              │
│ 多 Skill 冲突时触发，做出最终判定          │
├─────────────────────────────────────────┤
│ 第四层：Hermes Skill Squad (GPU, 按需)   │
│ 8 个审查员，各审一个维度，语义理解   │
├─────────────────────────────────────────┤
│ 第三层：结构化上下文匹配 (CPU)            │
│ 实体定位 + 段落归属 + 否定作用域 + 引用标识│
├─────────────────────────────────────────┤
│ 第二层：医学 NLP 预处理 (CPU)             │
│ 分词 + NER + 否定检测 + 段落解析          │
├─────────────────────────────────────────┤
│ 第一层：知识库质量清洗 (CPU)              │
│ 去多义词 + 加白名单 + 补排除模式          │
└─────────────────────────────────────────┘
```

---

## 第一层：知识库清洗

**解决盲区**：多义词（盲区三）

| 问题条目 | 修正 |
|----------|------|
| `"乳腺"` → gender 冲突 | 改为 `"乳腺癌"、"乳腺结节"` 等病理词 |
| `"附件"` → gender 冲突 | 改为 `"子宫附件"、"双侧附件区"` |
| `"出血"` → 危急值 | 改为 `"活动性出血"、"大量出血"` |
| `"气胸"` → 危急值 | 配置 `exclude_patterns: ["无.*气胸"]` |

规则配置改进：

```yaml
rules:
  - keywords: ["子宫肌瘤", "子宫内膜癌", "子宫腺肌症"]
    exclude_genders: ["男"]
    severity: critical
    exclude_patterns:
      - "未见.*子宫"
      - "子宫.*未见"
      - "子宫.*正常"
      - "子宫.*切除术后"
```

---

## 第二层：NLP 预处理

**解决盲区**：否定语境（盲区一）

新增 `NegationDetector`：逐句检测否定词（未见/无/未发现/排除/不考虑）→ 标记其作用域内所有实体的否定状态。

```
输入："子宫及双侧附件区未见显示（男性，正常骨盆）"
    ↓
实体：子宫[否定=true] 附件区[否定=true]
    ↓
规则匹配 → 命中但被否定 → 跳过 ✅
```

---

## 第三层：结构化上下文

**解决盲区**：引用/比较（盲区二）+ 部位重叠（盲区四）

1. 段落归属：识别 ClinicalHistory / Findings / Impression / Recommendation 段落
2. 规则作用域：性别矛盾 仅 Findings+Impression；平扫矛盾 仅 Findings（不包括 Recommendation）
3. 引用识别：标记"与前片比较""外院""建议"等模式 → 跳过对应规则

---

## 第四层：Hermes Skill Squad

**解决盲区**：多义词（三）+ 严重度（五）+ NLI 判定

每个 Skill 专注一个维度，不受其他维度干扰：
- `gender-anatomy-checker`：解剖-性别矛盾（不可调和时才报）
- `critical-sign-arbiter`：危急值审查（识别"陈旧性""术后""良性"等降级语境）
- `findings-impression-nli`：NLI 三分类替代粗糙余弦相似度

---

## 第五层：Hermes QA

**解决**：跨 Skill 冲突 + 边界 case

仅当多个 Skill 意见不一致、或 Skill 与规则引擎冲突时触发。

---

## 效果预期（假设备验证）

以下为基于根因分析的理论预期，需在每个 Phase 完成后用 200+ 回归报告实测验证：

| 层 | 预期误差降低 | 验证方法 | 验证时机 |
|----|:----------:|----------|----------|
| 第一层（知识库清洗） | 减少 10-15% 误报 | 对比清洗前后误报数 | Phase 1 完成 |
| 第二层（NLP 预处理） | 减少 20-30% 误报 | 否定检测精确率/召回率 | Phase 1 完成 |
| 第三层（结构化上下文） | 减少 10-15% 误报 | 段落归属准确率 + 引用识别准确率 | Phase 1 完成 |
| 第四层（Skill Squad） | 减少 15-25% 误报 | 每 Skill 独立 F1 + 端到端对比 | Phase 3 完成 |
| 第五层（Hermes QA） | 减少 5-10% 误报 | 冲突 case 中 QA 正确率 | Phase 3 完成 |
| **合计目标** | **从 ~40% → 5-10%** | 500+ 回归报告端到端评估 | Phase 3 结束 |

> 每层实际效果可能衰减（后续层处理的是余下误报，基数变小）。若某层未达预期，需回退到 `memory.md` 记录原因并更新策略。
