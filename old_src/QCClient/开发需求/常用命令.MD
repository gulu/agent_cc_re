# QCClient 常用命令

## 构建编译
```powershell
cd D:\admodel\risdev\ris\AI_QC-system\src\QCClient

# 还原包（仅在首次或添加包后需要）
dotnet restore

# 编译
dotnet build --project QCClient/QCClient.csproj

# 无还原编译（如果 restore 因 SDK bug 失败）
dotnet build --no-restore --project QCClient/QCClient.csproj
```

## 启动 QCClient
```powershell
cd D:\admodel\risdev\ris\AI_QC-system\src\QCClient
dotnet run --project QCClient/QCClient.csproj
```

启动后自动打开浏览器访问 Web 界面，控制台会打印访问地址。

## 测试后端连接

```powershell
# 测试 QCService (OCR + 报告查询)
curl "http://localhost:5200/api/v1/qc/query-report?accessNumber=81982632"

# 测试 ReportQC (质控)
curl "http://localhost:5100/api/v1/qc/reports/stats"

# 完整质控测试
$body = @{
    reportId = "TEST-81982632"
    findings = "胸廓对称，右肺可见多个径线0.3cm以下微结节..."
    impression = "1. 右肺微结节，建议年度复查"
    reportType = "CT"
    patientName = "孟秀华"
    patientGender = "女"
    patientAge = 64
} | ConvertTo-Json -Compress
curl "http://localhost:5100/api/v1/qc/report" -Method POST -Body $body -ContentType "application/json"
```

## 启动顺序

1. ReportQC (端口 5100) — 质控引擎
2. QCService (端口 5200) — OCR + 报告查询
3. QCClient (随机端口) — 客户端

## Web 界面

启动后自动打开浏览器，访问地址类似：
```
http://localhost:端口号
```
