## 一、项目背景

本项目为放射科报告智能质控系统，目前已完成**后端核心 API**（QCService+ReportQC）与**基础质控 Demo 页面**的开发。现需在`QCClient`目录下开发**轻量级托盘后台服务**，实现医生在日常报告书写过程中，**无感自动触发**报告质控，并通过嵌入式 Web 界面以侧边栏形式实时展示质控结果，全程不干扰医生正常工作流程，显著提升报告书写质量与审核效率。

## 二、架构决策（关键设计约定，强制执行）

### 2.1 整体架构：WinForms 托盘后台服务 + HttpListener 嵌入式 Web 服务器 + WebView2 嵌入面板

**最终确定架构**：.NET Framework 4.7.2 及以上 WinForms Application + HttpListener（内嵌轻量级 HTTP 服务器） + WebView2 嵌入静态 HTML/CSS/JS + 托盘气泡通知

> **注意**：本项目**不是** ASP.NET Core 应用，**不是** .NET (Core) 8 应用。必须使用 .NET Framework 的 WinForms 项目模板。

#### 设计核心理由

- **极致轻量**：服务进程常驻内存占用≤30MB，CPU 使用率≤1%（空闲时）。使用 HttpListener 而非完整 ASP.NET Core 可显著降低内存占用。
- **UI 无限灵活**：HTML/CSS/JS 可精确还原医疗系统设计规范，支持复杂交互与动画，通过 WebView2 嵌入 WinForms 窗口实现零浏览器干扰
- **不干扰工作流**：质控完成后先通过托盘气泡通知摘要信息，医生选择查看详情后才弹窗，不打断报告书写
- **窗口管理灵活**：WinForms 窗口可精确定位到屏幕右下角，可设置始终置顶，可通过 Ctrl+Q 快速隐藏/显示
- **零侵入性**：不修改医院现有 PACS / 报告系统，不安装任何插件，仅需运行一个可执行文件
- **易部署维护**：单文件发布，双击运行，自动升级，无需管理员权限
- **兼容性**：基于 .NET Framework 4.7.2+，Windows 10/11 内置运行时不需额外安装。WebView2 Runtime 在 Windows 11 中内置，Windows 10 需单独安装

### 2.2 通信架构：全后端代理模式（客户端零数据库依赖）

**绝对禁止**客户端直接连接任何数据库，所有数据交互 100% 通过后端 API 中转：

1. 客户端仅做三件事：**屏幕区域截图**、截图区域设置、**API 调用转发**、**Web 界面托管**
2. OCR 识别、报告查询、质控分析全部在后端服务器完成
3. 客户端与后端仅通过标准 HTTP/JSON 协议通信，无其他端口依赖

### 2.3 OCR 技术路线（最终确定）

- **OCR 引擎**：统一部署在后端 QCService 服务（已完成 Tesseract 5.2+chi_sim_fast 集成）
- **客户端职责**：仅根据配置的坐标区域截取屏幕图片，以 Base64 编码上传至后端
- **API 接口**：`POST /api/v1/ocr/recognize`，支持一次上传多个区域截图批量识别
- **优势**：统一 OCR 模型版本，便于后续升级优化，客户端无任何 OCR 依赖

### 2.4 核心业务流程（优化后）

```
1. 医生开机 → QCClient自动启动 → 最小化到系统托盘后台运行
   ↓
2. 医生打开PACS/报告系统开始书写报告
   ↓
3. QCClient后台定时（默认5秒）截取【检查流水号区域】→ 上传OCR识别
   ↓
4. 识别到新的有效影像号 → 自动触发完整质控流程
   ↓
5. 调用后端API查询报告数据 → 判断报告状态
   ├─ 报告已完成（reportContent+reportDiagnosis非空）→ 直接调用质控API
   └─ 报告正在书写（内容为空）→ 自动截取【报告所见】+【报告诊断】区域 → OCR提取内容
   ↓
6. 构造完整质控请求 → 调用ReportQC API执行分析
   ↓
7. 质控结果 → 托盘气泡通知（简洁显示：评分+通过/未通过）
   ↓
8. 医生查看气泡
   ├─ 双击气泡 → 打开WinForms浮动面板（WebView2加载HTML界面）显示完整详情
   ├─ 点击"打开质控面板"托盘菜单 → 打开浮动面板
   └─ 忽略气泡 → 气泡自动消失（3秒后）
   ↓
9. 浮动面板中查看问题列表与修正建议 → 修改报告 → 内容变化自动重新质控
   ↓
10. Ctrl+Q 快速隐藏/显示浮动面板
```

### 2.5 场景智能识别

- **书写报告场景**：OCR 持续监控报告内容区域，内容稳定 1秒后自动触发质控，每 30 秒最多触发一次
- **报告审核场景**：检测到影像号变化时立即触发一次完整质控
- **手动触发场景**：医生点击托盘菜单 "立即质控" 或 Web 界面 "重新质控" 按钮

## 三、技术约束（强制执行，违反视为不合格）

1. **开发框架**：
   - **运行时**：.NET Framework 4.7.2 或更高版本（禁止 .NET Core / .NET 5/6/7/8）
   - **项目类型**：Windows Forms 应用程序（`<OutputType>WinExe</OutputType>`）
   - **前端**：必须使用**纯 HTML/CSS/JS**，禁止引入任何前端框架（Vue/React/Angular 等）
   - **禁止使用任何第三方 UI 控件库**

2. **嵌入式 HTTP 服务器**：
   - 必须使用 `System.Net.HttpListener`（.NET Framework 内置）
   - **禁止**依赖 ASP.NET Core / Kestrel
   - **禁止**使用自宿主 ASP.NET（`Microsoft.AspNetCore.Server.Kestrel`）
   - 监听 `http://127.0.0.1:0`（随机端口，仅本地回环可访问）
   - 端口分配后将真实端口写入托盘菜单提示和日志

3. **JSON 序列化**：
   - 必须使用 `Newtonsoft.Json`（Json.NET）
   - **禁止**使用 `System.Text.Json`（.NET Framework 不内置）
   - NuGet 包：`Newtonsoft.Json` 最新稳定版

4. **日志系统**：
   - 推荐使用 `Serilog`（.NET Framework 版，NuGet：`Serilog` + `Serilog.Sinks.File` + `Serilog.Sinks.Console`）
   - 允许使用 `NLog` 作为替代方案
   - **禁止**使用 ASP.NET Core 日志抽象（`Microsoft.Extensions.Logging`）

5. **屏幕截图**：
   - 使用 `System.Drawing.Graphics.CopyFromScreen()`（.NET Framework 内置）
   - 依赖 NuGet 包：无（`System.Drawing` 为 .NET Framework 内置程序集）

6. **SSE（Server-Sent Events）**：
   - 通过 `HttpListenerResponse` 手动实现 SSE 推送
   - **禁止**依赖 ASP.NET Core 的 SSE 中间件
   - 格式：`data: {json}\n\n`

7. **依赖注入**：
   - **禁止**使用 ASP.NET Core 的 `IServiceProvider` / `IServiceCollection`
   - 使用手动构造或简单的服务定位器模式

8. **异步编程**：
   - 使用 `async/await`（.NET Framework 4.5+ 支持）
   - HttpListener 的异步版本：`GetContextAsync()`
   - 注意不要使用 .NET Core 独有的 API（如 `IAsyncEnumerable<T>`）

9. **WebView2 嵌入**：
   - 使用 `Microsoft.Web.WebView2` NuGet 包
   - 嵌入 WinForms 浮动面板（`QcPanelForm.cs`），定位在屏幕右下角（Bottom-Right）
   - WebView2 加载本地 HttpListener 的 `http://127.0.0.1:{port}/` 页面
   - 宽度固定 320px，高度自适应（屏幕高度 - 任务栏高度）
   - 设置 `AlwaysOnTop = true`（可在配置中关闭）
   - 如果 WebView2 Runtime 未安装，自动打开系统浏览器作为降级方案
   - **禁止**使用 WinForms 内置的 `WebBrowser` 控件（IE 内核，不支持现代 CSS）

10. **托盘气泡通知**：
   - 使用 `NotifyIcon.ShowBalloonTip()`（.NET Framework 原生支持）
   - 质控完成后立即弹出气泡通知（3600ms 显示时长）
   - 通知内容：质控评分 + 通过/未通过状态 + 摘要
   - 点击/双击气泡 → 打开浮动面板查看详情

## 四、核心功能需求

### 4.1 托盘服务核心功能

#### 4.1.1 托盘图标与菜单

- 系统托盘显示程序图标，鼠标悬停显示 "放射科报告质控助手"
- 右键菜单包含以下选项：
  - 打开质控面板（点击左键也打开）
  - 立即质控
  - 开机自启动（勾选 / 取消）
  - 设置
  - 关于
  - 退出

#### 4.1.2 自动启动与后台运行

- 支持设置开机自启动（无需管理员权限，写入当前用户注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`）
- 启动后自动最小化到系统托盘，不显示主窗口
- 禁止重复启动，检测到已有实例运行时激活现有实例并打开 Web 界面

#### 4.1.3 多 OCR 区域配置与管理

- 支持配置**任意多个** OCR 识别区域，每个区域包含：
  - 区域名称（如：检查流水号、报告所见、报告诊断）
  - 区域类型（枚举：`access_number`/`report_content`/`report_diagnosis`）
  - 屏幕坐标（X,Y,Width,Height）
  - 识别频率（秒）
  - 是否启用
- 提供 Web 界面可视化区域配置工具：
  - 点击 "选择区域" 按钮 → 屏幕变暗 → 鼠标拖动框选
  - 实时显示区域坐标和大小
  - 支持保存 / 加载多套配置方案（适配不同分辨率 / 不同 PACS 系统）

#### 4.1.4 自动触发逻辑

- 后台定时任务监控所有启用的 OCR 区域
- **触发条件**：当`access_number`区域识别到新的、格式正确的检查流水号时
- **防抖处理**：同一影像号 5 分钟内不重复触发质控
- **内容变化检测**：报告内容区域文字变化超过 10% 时自动重新触发质控

### 4.2 报告数据查询与质控流程

1. **报告查询**：通过影像号调用后端`GET /api/v1/qc/query-report?accessNumber={no}`
2. **数据判断**：
   - 正常流程：`reportContent`和`reportDiagnosis`均非空 → 直接使用
   - 书写中流程：任一为空 → 自动截取`report_content`和`report_diagnosis`区域 → OCR 识别 → 合并到请求数据
3. **质控调用**：构造完整请求体，调用`POST /api/v1/qc/report`
4. **结果推送**：通过 SSE 将质控结果实时推送到所有连接的 Web 客户端

### 4.3 嵌入式浮动面板（替代浏览器侧边栏）

**界面定位**：WinForms 浮动面板（`QcPanelForm`），内嵌 WebView2 加载本地静态 HTML/CSS/JS 界面。面板固定宽度 320px，高度自适应屏幕（任务栏上方），**默认停靠在屏幕右下角**，始终置顶显示，不干扰医生正常报告书写工作。

> 与传统的浏览器侧边栏方案相比，WinForms 嵌入方案的优势：
> - 不打开浏览器，不干扰医生工作界面
> - 可以精确定位窗口位置（右下角弹出）
> - 支持 Ctrl+Q 快捷键快速隐藏/显示
> - 可以设置任务栏不显示图标，减少干扰
> - 通过 WebView2 加载本地 HTTP 服务，保持 HTML/CSS/JS 的灵活性

- 通过 SSE 与嵌入式 HTTP 服务器保持长连接

- 质控进度实时推送（截图中→识别中→查询中→分析中→完成）
- 连接断开时自动重连，重连成功后自动恢复上次状态

### 4.4 手动功能

- 支持手动输入影像号触发质控
- 支持扫描枪输入影像号（自动识别回车结束符）
- 支持手动粘贴报告内容进行质控（OCR 识别效果不佳时）
- 支持查看最近 20 条质控历史记录

### 5.2 日志系统

- 使用 Serilog 作为日志框架（.NET Framework 兼容版本）
- NuGet 包：`Serilog`、`Serilog.Sinks.File`、`Serilog.Sinks.Console`
- 同时输出到控制台和文件
- 日志文件按天滚动，保留最近 30 天
- 日志内容包含：时间、级别、模块、详细信息、异常堆栈
- 敏感信息（患者姓名、身份证号）必须脱敏处理

## 六、异常处理与容错

1. **后端连接失败**：
   - 记录详细错误日志
   - Web 界面显示离线状态
   - 每 30 秒自动重试连接
   - 不弹窗打扰用户

2. **OCR 识别失败**：
   - 记录日志
   - 继续下一次识别
   - 连续失败 5 次后在 Web 界面显示提示

3. **截图失败**：
   - 记录日志
   - 跳过本次识别
   - 连续失败 10 次后自动重启服务

4. **程序崩溃**：
   - 记录崩溃日志
   - 自动重启服务（最多重启 3 次 / 小时）

### 8.1 关键 API 路由设计

HttpListener 嵌入式 HTTP 服务器，仅对内提供静态文件服务和 SSE/API 接口：

| HTTP 方法 | 路径 | 功能 |
|-----------|------|------|
| GET | `/api/status` | 获取后端连接状态 |
| POST | `/api/qc` | 手动触发质控 |
| POST | `/api/qc/manual` | 手动输入质控 |
| GET | `/api/sse` | SSE 推送连接 |
| GET | `/api/config` | 获取完整配置 |
| POST | `/api/config/ocr-areas` | 更新 OCR 区域配置 |
| GET | `/api/config/web` | 获取 Web 设置 |
| POST | `/api/config/web` | 更新 Web 设置 |
| POST | `/api/config/reset` | 恢复默认配置 |
| GET | `/` 或静态文件 | 提供 wwwroot 静态文件 |
